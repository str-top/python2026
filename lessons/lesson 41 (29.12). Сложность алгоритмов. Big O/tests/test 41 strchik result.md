```python
# Пример сортировки
def bubble_sort(arr):
    n = len(arr)
    for i in range(n):
        for j in range(0, n-i-1):
            if arr[j] > arr[j+1]:
                arr[j], arr[j+1] = arr[j+1], arr[j]
```

## Графы

Какова временная сложность алгоритма поиска в глубину (DFS) в графе с V вершинами и E рёбрами?

```python
def dfs(graph, start):
    visited = set()
    stack = [start]
    while stack:
        vertex = stack.pop()
        if vertex not in visited:
            visited.add(vertex)
            stack.extend(graph[vertex] - visited)
    return visited
```
  
Варианты ответов:

- [ ] Option 4: O(E log V) - зависит от рёбер и вершин

#  Вопрос 6. Амортизированный анализ
  
## Амортизированный анализ

Объясните концепцию амортизированного анализа на примере динамического массива (list в Python), когда он увеличивает свой размер.
  
Answer:
Амортизированный анализ учитывает среднюю стоимость операции в последовательности операций, а не стоимость одной операции в худшем случае.

В случае динамического массива (Python list):
	•	Когда массив заполнен, добавление нового элемента требует увеличения размера массива (обычно в 2 раза), что копирует все элементы в новый массив. Это дорого: O(n).
	•	Но такие увеличения происходят редко. Большинство добавлений выполняются за O(1).

Амортизированная стоимость добавления элемента = O(1), потому что если посчитать все дорогостоящие копирования за весь набор операций, средняя стоимость одной вставки остаётся постоянной.

Отличное объяснение! Вы верно описали амортизированную сложность динамического массива, привели примеры и сделали акцент на средней стоимости операций. Полный балл.

```python

    if n <= 1:
        return n
    dp = [0, 1]
    for i in range(2, n+1):
```
